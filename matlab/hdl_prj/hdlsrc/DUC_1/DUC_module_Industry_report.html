CGSL-2.E.A.4, (参考),   标识由超出指定数量的驱动器驱动的三态总线。
CGSL-2.E.A.5a, (默认),   除三态网络以外的网络不能由多个驱动器驱动。
CGSL-2.E.A.6, (强制),   Inout 端口不应直接连接到输入/输出。
CGSL-2.E.A.7, (推荐 2),   不要在 if 语句的条件表达式中使用三态输出。
CGSL-2.E.A.8, (推荐 2),   对于在 others 选项中赋予固定值的 case 语句，不应在该语句的选择表达式中使用三态输出。
CGSL-2.E.A.9, (默认),   不要在 casex 和 casez 语句的选择表达式中输入三态输出。
CGSL-2.E.B   连接约束
CGSL-2.E.B.1, (推荐 2),   由三态网络直接驱动的逻辑应置于单独的模块中。
CGSL-2.F   考虑电路结构的 always/process 构造规范
CGSL-2.F.B   条件语句数量约束

在单个 process 块中不要单独描述超过 1 个语句(if/case/while/for/ 循环)；但在 "firRdy" 块中找到 8 个语句。   DUC_1/DUC_module/LP_system/Lowpass/FirRdyLogic
在单个 process 块中不要单独描述超过 1 个语句(if/case/while/for/ 循环)；但在 "InputControl" 块中找到 10 个语句。   DUC_1/DUC_module/LP_system/Lowpass
CGSL-2.F.B.1a, (强制),   不要在多个级联条件(if/case)区域中写入同一寄存器/线网。
CGSL-2.F.B.2, (强制),   敏感列表中的一个变量在同一 always 块内被修改。
CGSL-2.G   "IF" 语句描述规范

CGSL-2.G.C.1c, (默认),   if...else if 构造的链不能超出 7 层

CGSL-2.H   "CASE" 语句描述规范
CGSL-2.H.A   CASE 结构约束
CGSL-2.H.A.3, (参考),   case 构造不应有重叠的子句条件。
CGSL-2.H.A.5, (参考),   不要使用 full_case 指令。
CGSL-2.H.C   默认值约束
CGSL-2.H.C.3, (推荐 1),   当并非所有条件都描述为 case 子句或缺失 default 子句时，不要使用 //synopsys full_case pragma。
CGSL-2.H.C.4a, (默认),   不应在 if 条件中使用在 case default 子句中被赋予 don't care 值的信号。
CGSL-2.H.C.4b, (默认),   不应在 "ternary" 和 "case" 构造中使用在 case default 子句中被赋予 don't care 值的信号。
CGSL-2.H.C.5, (推荐 1),   case 构造中的 default 子句必须为最后一个子句。
CGSL-2.H.C.6, (默认),   对于未在 default 子句中赋 'X' 值的 case 语句，不要在其选择表达式中使用被赋予 don't care 值的信号。
CGSL-2.H.C.7, (默认),   不要在 casex 语句的选择表达式中使用被赋予 don't care 值的信号。
CGSL-2.H.D   don't Care 约束
CGSL-2.H.D.1a, (默认),   casex 或 casez 构造必须包含一个带 don't care 值的条件
CGSL-2.H.D.1b, (默认),   casex 或 casez 分支中的 don't care 条件的编码风格不正确
CGSL-2.H.D.3, (默认),   设计不应使用 casex 或 casez 构造。
CGSL-2.H.D.4, (默认),   casex 或 casez 构造不能有复杂的子句条件。
CGSL-2.H.E   附加 CASE 约束
CGSL-2.H.E.1, (默认),   不要使用 parallel_case 指令。
CGSL-2.H.E.2, (默认),   不要在 case 构造条件中使用固定值。
CGSL-2.H.E.3, (默认),   不要使用变量或表达式作为 case 子句条件。
CGSL-2.H.E.4, (默认),   case 构造条件中不应使用逻辑、算术、按位或归约运算。
